Original Article: https://www.bleepingcomputer.com/news/security/fortiweb-flaw-with-public-poc-actively-exploited-to-create-admin-users/

1) The article discusses a vulnerability in Fortinet FortiWeb devices that allows threat actors to create unauthorized admin users through a path traversal flaw. This exploit, observed initially by Defused, involves sending HTTP POST requests to a specific endpoint, resulting in the creation of accounts with admin privileges. The flaw, affecting versions 8.0.1 and earlier, has prompted threat actors to execute global attacks after a detailed proof of concept became public. Security researchers and the company Fortinet are actively investigating this incident that significantly impacts system security integrity.

2) This vulnerability is important because it compromises FortiWeb devices, a product used to protect web applications and APIs. Unauthorized admin access undermines security defenses, potentially exposing sensitive data and networks to further attacks once threat actors exploit it. The public disclosure of the exploit, coupled with the rise in global attack attempts, underscores the critical need for keeping security devices updated and monitoring unusual activity. Hence, it signals organizations to scrutinize their cybersecurity defenses and response strategies continuously.

3) Organizations using FortiWeb should act by promptly updating to version 8.0.2, which addresses the flaw. Administrators are advised to review their systems for any unauthorized admin accounts and scrutinize logs for suspicious activity linked with the vulnerability's exploit paths and the noted IP addresses. Limiting network access by ensuring management interfaces are not exposed to the internet and allowing connections through trusted networks or VPNs is also prudent. These measures, combined with increased monitoring for anomalies, can help mitigate the risk posed by the vulnerability.
