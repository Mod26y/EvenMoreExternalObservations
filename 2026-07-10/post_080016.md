Original Article: https://www.bleepingcomputer.com/news/security/injective-sdk-on-npm-infected-with-cryptocurrency-wallet-stealer/

1) The breach involved hackers compromising the GitHub repository of the Injective Labs SDK project to publish a malicious package on npm. This package targeted cryptocurrency wallet private keys and mnemonic seed phrases. Security firms Socket, Ox Security, and StepSecurity identified this supply-chain attack with version 1.20.21 of the package and reported thousands of downloads and dependencies that became potential vectors for further compromise.

2) This incident highlights the growing risks associated with supply-chain attacks, particularly in the dynamic cryptocurrency and decentralized finance (DeFi) sectors. With over 50,000 weekly downloads, the compromised package posed a significant threat to any systems utilizing the infected SDK. Stolen private keys and seed phrases could result in the unauthorized transfer and loss of digital assets, reflecting the critical need for robust security practices in software development.

3) In response, developers and organizations should review and enhance their supply-chain security protocols by implementing rigorous checks on package sources and monitoring for suspicious activity. Systems that may have interacted with the compromised package should be investigated, and any affected keys or seed phrases should be rotated. Educating developers on recognizing and responding to such vulnerabilities could help prevent future incidents. Lastly, continuous vulnerability scanning and utilization of breach and attack simulation tools can foster a proactive security posture.
