Original Article: https://www.bleepingcomputer.com/news/security/-asyncapi-npm-packages-infected-with-credential-stealing-malware/

1) **What happened:**  
The cybersecurity incident involved a supply-chain attack on the Node Package Manager (npm) affecting five malicious versions of the AsyncAPI packages. The attacker exploited misconfigured GitHub Actions workflows to distribute a remote access trojan with information-stealing capabilities. By compromising two AsyncAPI GitHub repositories, the threat actor injected malware, leading to the publication of trojanized packages with over 2.25 million weekly downloads. The attack leveraged GitHub's integrated release workflows to ensure the packages appeared legitimate, infiltrating the supply chain without exploiting stolen tokens or involving malicious maintainers.

2) **Why it matters:**  
This incident highlights the growing threat of supply-chain attacks, where attackers infiltrate legitimate development processes to spread malware widely. Such breaches can compromise the security of thousands of systems, given the high download volume of affected npm packages. The attack underscores vulnerabilities within software development workflows, including CI/CD pipelines, which can impact downstream customers significantly. This type of attack exploits trust in widely used tools, posing risks of credential theft and unauthorized access to sensitive data. Understanding the mechanisms used can help organizations fortify their defenses against similar attacks.

3) **What actions should be taken:**  
Organizations should perform a detailed audit of potentially impacted systems by identifying if any compromised AsyncAPI packages were installed during the exposure period. Actions include pinning dependencies to known-good versions, regenerating lock files, and purging the malicious 'NodeJS/sync.js' payload. It would be prudent to terminate all suspicious processes actively and rotate any potentially compromised credentials. Moreover, enhancing workflow security controls, such as implementing stringent access controls on CI/CD pipelines and regularly auditing workflows, can help mitigate similar risks. Continuous monitoring and breach simulation testing can strengthen detection and response capabilities for future incidents.
