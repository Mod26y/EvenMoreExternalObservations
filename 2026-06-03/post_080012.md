Original Article: https://www.bleepingcomputer.com/news/security/vs-code-zero-day-lets-hackers-steal-github-tokens-in-one-click/

1) What happened:
A zero-day vulnerability in Visual Studio Code (VS Code) has been disclosed, allowing attackers to steal GitHub OAuth tokens when a user clicks a malicious link. This exploit manipulates the message-passing system between the browser and the VS Code editor, enabling the execution of malicious JavaScript code. This code can install extensions that access OAuth tokens, which have broad GitHub access. Researcher Ammar Askar disclosed this vulnerability publicly after an unsatisfactory prior experience with Microsoft's response process and a lack of acknowledgment in handling such security issues.

2) Why it matters:
This vulnerability is significant because it leverages widely used software and popular developer tools like VS Code and GitHub, which host numerous code repositories, including sensitive and proprietary projects. Access to OAuth tokens could allow attackers to compromise all repositories a user has access to, risking intellectual property, security configurations, and potentially introducing malicious code. This incident also highlights challenges in vulnerability disclosure processes, impacting trust between security researchers and software vendors like Microsoft.

3) What actions should be taken:
Users of VS Code should take immediate preventive actions by clearing browser cookies and site data for github.dev to ensure warnings for suspicious extension sign-ins are enabled. Organizations should educate developers about phishing and suspicious link risks and employ security solutions that monitor and restrict unauthorized code executions. Additionally, it's prudent for Microsoft's security response policies to incorporate collaborative communication with researchers to address vulnerabilities effectively and maintain trust within the cybersecurity community. Keeping software environments and permissions audited to mitigate risk exposures stemming from such vulnerabilities is also important.
