Original Article: https://www.bleepingcomputer.com/news/security/decades-old-finger-protocol-abused-in-clickfix-malware-attacks/

The article discusses how cybercriminals are exploiting the Finger protocol—a legacy function that retrieves user information on Unix, Linux, and Windows systems—to execute malicious commands remotely. They are leveraging this often-overlooked protocol to run ClickFix malware attacks on Windows devices by instructing victims to execute commands impersonating system verifications. The abuse of such an outdated protocol highlights how attackers constantly seek unconventional methods to exploit systems.

This development is significant because it illustrates the adaptive nature of cyber threats; by using the under-utilized Finger protocol, attackers bypass common detection methods. The attacks include tactics like impersonating human verification processes, making them more deceptive. As older protocols like Finger still exist in some infrastructures, organizations may overlook them as potential attack vectors, presenting a hidden vulnerability that requires attention.

Organizations could focus on proactive monitoring of seldom-used protocols like Finger to identify potential abuse points. Implementing network controls, such as blocking outbound traffic on TCP port 79, can prevent such attacks. Regularly updating and auditing systems, along with cybersecurity training to recognize social engineering tactics, can mitigate risks. Security teams should stay informed on emerging threats and consider integrating legacy protocol monitoring into their cybersecurity strategy.
