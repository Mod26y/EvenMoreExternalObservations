Original Article: https://www.bleepingcomputer.com/news/security/new-polyshell-flaw-allows-unauthenticated-rce-on-magento-e-stores/

1) The "PolyShell" vulnerability impacts all installations of Magento Open Source and Adobe Commerce version 2, potentially allowing unauthenticated remote code execution (RCE) and account takeovers. The flaw is rooted in Magento's REST API, which mishandles file uploads by storing them unsafely, thus enabling malicious scripts to be executed on servers. Although this vulnerability has not yet been exploited in real-world attacks, the exploit method is reportedly already publicized, and experts anticipate automated attacks imminently. Adobe has released an initial fix in an alpha version, but it is not available for production systems, leaving most users temporarily vulnerable.

2) The PolyShell flaw is critical as it jeopardizes the security of online stores running Magento, a widely used e-commerce platform. This vulnerability allows attackers to take control of affected stores, leading to data breaches and financial losses, and damaging customer trust and brand reputation. As these installations are integral to business operations worldwide, the potential for distributed, automated attacks increases the urgency for swift remediation. Furthermore, the flaw highlights systemic vulnerabilities in custom server setups that many e-commerce businesses use, underscoring the need for robust configuration management.

3) In response to the PolyShell vulnerability, administrators of affected Magento stores should implement temporary protective measures. They can restrict access to the file upload directory (pub/media/custom_options/) and verify current web server configurations with nginx or Apache to prevent unauthorized file access. Regular scans of stores for malware, such as backdoors or uploaded malicious shells, are advised to mitigate potential damage until a production-ready fix is available from Adobe. Additionally, store operators should maintain communication with Adobe for updates on the official security patch release to ensure they remain protected in the long term.
