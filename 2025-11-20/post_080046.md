Original Article: https://www.bleepingcomputer.com/news/security/w3-total-cache-wordpress-plugin-vulnerable-to-php-command-injection/

The W3 Total Cache WordPress plugin was found to have a critical PHP command injection flaw, CVE-2025-9501, which could allow an attacker to execute arbitrary PHP commands on the server by simply posting a malicious comment. This issue affects versions before 2.8.13 and poses a significant risk since W3TC is used on over a million websites. Despite an update being available, a large number of sites remain vulnerable, based on download statistics that suggest inadequate patch uptake.

This vulnerability is significant because it provides potential attackers with a simple avenue to gain unauthorized access and possibly full control of a website. Since W3 Total Cache is incredibly popular for performance optimization, the widespread exposure to this flaw could result in a major security incident impacting numerous sites worldwide. Rapid exploitation is anticipated once a public exploit is released, highlighting the urgency for affected users to act promptly.

Organizations should prioritize updating to version 2.8.13 of W3 Total Cache if they use the plugin to eliminate the risk. For those unable to update immediately, temporary solutions include deactivating the plugin or adjusting comment settings to block malicious input. Regularly auditing and updating plugins, especially those with known vulnerabilities, is crucial to maintaining a secure WordPress environment. Moreover, organizations should keep abreast of forthcoming proof-of-concept exploit releases to manage their risk exposure efficiently.
