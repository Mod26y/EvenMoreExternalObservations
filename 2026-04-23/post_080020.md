A new Mirai-based malware campaign is exploiting a command-injection vulnerability, CVE-2025-29635, in D-Link DIR-823X routers, allowing remote command execution. The exploit involves sending a POST request to a vulnerable endpoint, leading to the installation of a malware named "tuxnokill," which facilitates DDoS attacks. Although disclosed over a year ago, this is the first observed active exploitation, using techniques previously identified in proof-of-concept exploits. This campaign also targets older routers from TP-Link and ZTE, exploiting similar vulnerabilities across these devices.

The significance of this campaign lies in its use of a known vulnerability in end-of-life (EoL) routers, which might be prevalent due to ongoing use despite a lack of security support from the manufacturer. Exploiting such unpatched routers puts numerous internet-connected devices at risk, potentially turning them into botnet nodes used for disruptive DDoS attacks. This underlines the critical importance of upgrading outdated hardware or ensuring mitigations are in place to protect against these vulnerabilities.

As a response, affected users are advised to upgrade to newer router models actively supported and receiving security patches. Disabling remote administration portals, changing default passwords, and closely monitoring router configurations could reduce exploitation risks. Administrators overseeing such devices should regularly audit their network for security weaknesses, ensuring vulnerabilities in unsupported hardware do not compromise overall network integrity.
