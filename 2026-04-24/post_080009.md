Original Article: https://www.bleepingcomputer.com/news/security/hackers-exploit-file-upload-bug-in-breeze-cache-wordpress-plugin/

1) The incident involves hackers exploiting a critical vulnerability identified as CVE-2026-3844 in the Breeze Cache plugin for WordPress. This bug permits the unauthorized upload of arbitrary files to a server, potentially leading to remote code execution and full website control. The issue originated from missing file-type validation in the plugin's 'fetch_gravatar_from_remote' function, affecting versions up to 2.4.4. The vulnerability becomes active when the “Host Files Locally - Gravatars” add-on is enabled, though it is not enabled by default. The flaw has been patched in version 2.4.5.

2) This vulnerability is significant because it threatens the security of over 400,000 websites using the Breeze Cache plugin, which plays a crucial role in enhancing performance by managing caching and optimizing files. A critical severity score of 9.8 underscores the risk of remote code execution and complete site takeover. Prompt exploitation attempts suggest potential widespread attacks if site owners fail to act. For businesses and individuals relying on WordPress, this presents a tangible risk to data integrity, service availability, and website reputation.

3) As a response, web administrators should immediately update the Breeze Cache plugin to version 2.4.5 to mitigate vulnerability risks. If immediate upgrading isn't feasible, turning off the “Host Files Locally - Gravatars” add-on serves as a temporary protective measure. Regular monitoring of WordPress installations and reinforcing security practices, such as regular updates and vulnerability assessments, remains crucial in preventing similar threats. Users should increase awareness about plugin settings and dependencies that may unintentionally expose them to vulnerabilities.
