Original Article: https://www.bleepingcomputer.com/news/security/critical-juniper-networks-ptx-flaw-allows-full-router-takeover/

1) The article reports a critical vulnerability (CVE-2026-21902) in Junos OS Evolved used on Juniper Networks PTX Series routers. The flaw is due to incorrect permission assignment in the ‘On-Box Anomaly Detection’ framework, allowing unauthorized access through an external port. This could lead to remote code execution with root privileges, potentially enabling attackers to fully take over the device. The issue affects versions before 25.4R1-S1-EVO and 25.4R2-EVO, but is not observed in standard Junos OS or in versions that are no longer supported.

2) This vulnerability is significant as PTX Series routers are deployed in critical infrastructure by ISPs, telecommunications, and cloud networks. An exploit could compromise network integrity, service reliability, and potentially lead to extensive data breaches or facilitate subsequent attacks. Given the strategic importance of these routers in high-bandwidth environments, they present a lucrative target for cyber-espionage or malicious cyber operations, as evidenced by past incidents involving Juniper products.

3) Organizations using the affected Junos OS Evolved versions should prioritize updating to fixed versions 25.4R1-S1-EVO, 25.4R2-EVO, or 26.2R1-EVO as soon as possible. Meanwhile, access to the vulnerable endpoints should be restricted to trusted networks using firewall filters or ACLs. Alternatively, the vulnerable service can be disabled to mitigate risk. Staying informed on patches and security advisories from Juniper and maintaining a robust incident response plan will be essential to preempt potential exploitation of similar vulnerabilities.
