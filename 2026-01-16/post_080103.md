Original Article: https://www.bleepingcomputer.com/news/security/hackers-exploit-modular-ds-wordpress-plugin-flaw-for-admin-access/

1) The article reports that a severe vulnerability in the Modular DS WordPress plugin (CVE-2026-23550) is being exploited by hackers. The flaw allows attackers to bypass authentication and gain admin-level access to sites using this plugin. This affects versions 2.5.1 and older, with active exploitation starting on January 13. Patchstack researchers identified the vulnerability, which results from inadequate checks on request origins when “direct request” mode is activated, allowing unauthorized privilege escalation. A fix was quickly released in version 2.5.2 to address these issues.

2) This vulnerability is a critical security concern as it compromises over 40,000 installations of the Modular DS plugin. Unauthorized admin access could lead to data breaches, website defacements, and deeper intrusions across interconnected WordPress sites managed through this plugin. The ability for malefactors to escalate privileges without prior authentication amplifies the risk, making it imperative for site administrators to address this flaw swiftly. The broader implications stress the importance of ongoing vigilance and timely patching in the WordPress ecosystem to prevent significant disruptions and data loss.

3) Users of the Modular DS plugin should upgrade to version 2.5.2 or later immediately to mitigate the risk of exploitation. It is advisable to review server access logs for suspicious activities and verify admin user accounts for unauthorized additions. Regenerating all WordPress salts can further enhance security posture post-update. Periodic security reviews and implementing strong access controls and monitoring are beneficial preventive strategies. Staying informed through vendor security bulletins and applying patches promptly can avoid potential compromises from emerging threats.
