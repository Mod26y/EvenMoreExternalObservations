Original Article: https://www.bleepingcomputer.com/news/security/hackers-exploit-auth-bypass-flaw-in-burst-statistics-wordpress-plugin/

1) **What happened:** Hackers are exploiting a critical authentication bypass vulnerability, CVE-2026-8181, in the Burst Statistics WordPress plugin. This flaw allows attackers to impersonate admin users and create rogue admin accounts without prior authentication. The vulnerability arose due to misinterpretation of authentication function results in the plugin's code from versions 3.4.0 and 3.4.1, which were released in April 2026. Attackers can access admin functions and databases, plant backdoors, and distribute malware. Wordfence reported over 7,400 attacks targeting this flaw within a 24-hour period.

2) **Why it matters:** The exploitation of this vulnerability is significant due to the potential for vast unauthorized control over websites, jeopardizing sensitive data and the integrity of affected sites. As the Burst Statistics plugin is active on 200,000 websites, particularly for those that handle private or critical information, the impact can be widespread. As attackers gain admin-level access, they can disrupt operations, tarnish reputations, and cause financial loss by distributing malware or redirecting users to malicious sites.

3) **What actions should be taken:** Users of the Burst Statistics plugin should immediately upgrade to the patched version 3.4.2 or disable the plugin to mitigate potential security breaches. Additionally, administrators should routinely monitor and restrict API access, employ strong and unique administrator credentials, and consider deploying comprehensive security solutions like Web Application Firewalls (WAFs) to detect and block malicious traffic. Running regular security audits and keeping all WordPress plugins and themes updated are also crucial preventive measures.
