Original Article: https://www.bleepingcomputer.com/news/security/github-announces-npm-security-changes-to-tackle-supply-chain-attacks/

1) What happened: GitHub has announced upcoming security changes in npm v12, designed to counter supply-chain attacks targeting the 'npm install' command. The changes will require explicit approval for code executions and the installation of non-registry and remote dependency sources, which previously occurred automatically. This initiative aims to close off vectors that attackers could exploit during automated dependency installations, such as preinstall and postinstall script fraud and abuse of Git and remote URL dependencies.

2) Why it matters: This development is crucial as it addresses significant supply-chain vulnerabilities exploited in recent hacks, like the distribution of malicious scripts via npm package scripts and Git dependencies. By requiring explicit permissions, these changes effectively mitigate attack vectors that could potentially compromise numerous projects relying on npm for dependency management during CI/CD procedures. This proactive approach could significantly curtail incidents like the Shai-Hulud attacks and other malicious campaigns, thereby enhancing overall software supply chain security.

3) What actions should be taken: Organizations and developers should upgrade to npm 11.16.0 or newer to view warnings for actions that will break with the new version. This preparatory step allows them to identify dependencies and workflows necessitating adjustments before migrating to npm v12. Participation in the community discussion is encouraged to provide feedback that might aid the transition. For security teams, continuous readiness such as implementing effective breach and attack simulation tactics to assess SIEM and EDR efficacy against these evolving threats is advised.
