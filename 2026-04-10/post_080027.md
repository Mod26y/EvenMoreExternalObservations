Original Article: https://www.bleepingcomputer.com/news/security/google-chrome-adds-infostealer-protection-against-session-cookie-theft/

**1) What happened:**

Google has introduced Device Bound Session Credentials (DBSC) protection in Chrome 146 for Windows. This enhancement aims to prevent info-stealing malware from obtaining and misusing session cookies. The protection binds a session’s data to a user’s hardware via cryptographic keys generated by security chips like TPM on Windows or Secure Enclave on macOS. These keys prevent the export of sensitive session data, rendering stolen session cookies useless. macOS users will receive this feature in a subsequent release, further enhancing security against advanced infostealer malware targeting browser sessions.

**2) Why it matters:**

Session cookies act as authentication tokens facilitating access to online services without repeatedly entering passwords. If stolen, these cookies allow attackers to impersonate legitimate users, leading to potential unauthorized access to accounts. Advanced infostealer malware has become adept at harvesting these cookies, posing significant security risks. The introduction of DBSC represents a noteworthy improvement in securing user sessions on online platforms, thereby strengthening overall cybersecurity resilience. By tying session data to specific devices, DBSC limits the usability of exfiltrated cookies, significantly reducing the efficacy of such attacks.

**3) What actions should be taken as a result of this information:**

Organizations should evaluate their web application security strategies in light of DBSC protocol advancements. IT departments should ensure their systems are updated with the latest Chrome version to benefit from these security enhancements. Web developers are encouraged to update their sites to support hardware-bound sessions by integrating DBSC-compatible endpoints. Security teams should also educate users about browser security updates and their importance. Additionally, engaging with guides and specifications such as those provided by Google and W3C will facilitate the seamless adoption of DBSC, fortifying defenses against session cookie theft.
