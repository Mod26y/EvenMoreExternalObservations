Original Article: https://www.bleepingcomputer.com/news/security/new-gobruteforcer-attack-wave-targets-crypto-blockchain-projects/

**1) What happened:**

A new attack wave involving the GoBruteforcer botnet targets servers hosting cryptocurrency and blockchain projects. This botnet exploits weakly configured databases, often via AI-generated examples, by launching brute-force login attacks. The malware uses compromised Linux servers to scan random IPs, leveraging weak default passwords in services like FTP, MySQL, PostgreSQL, and phpMyAdmin. Once access is gained, a web shell is uploaded, leading to further infections. GoBruteforcer is aided by outdated server stacks and predictable usernames from AI-generated content, targeting exposed servers and unleashing up to 95 brute-forcing threads to compromise systems.

**2) Why it matters:**

This attack wave is significant because it exploits vulnerabilities stemming from modern AI deployment trends and outdated server configurations, directly impacting the cryptocurrency and blockchain sectors. With 50,000 internet-facing servers identified as potentially vulnerable, there is a substantial risk to data integrity and financial assets. The use of AI-generated default configurations makes numerous systems susceptible, amplifying the impact and reach of these attacks. This highlights the critical need for secure configuration management and updated deployments to protect sensitive digital assets in increasingly popular decentralized digital economies.

**3) What actions should be taken as a result of this information:**

Administrators should actively avoid using AI-generated deployment guides that propose default or weak usernames and passwords. Implementing strong, unique credentials and regularly updating server software, particularly for FTP, phpMyAdmin, MySQL, PostgreSQL, and obsolete systems like XAMPP, is vital. Conduct thorough network scans for exposed services and reconfigure systems to tighten security settings. Additionally, adopting more secure, updated alternatives to outdated server stacks is recommended. Continuous monitoring and auditing can prevent potential breaches, while disseminating security awareness regarding these AI-driven vulnerabilities can help mitigate risks.
