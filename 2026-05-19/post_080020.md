The new SHub macOS infostealer, called Reaper, imitates Apple security updates using AppleScript to trick victims. It installs a backdoor, allowing the attacker to steal sensitive browser data, documents, financial details, and hijack crypto wallets. Users are deceived through fake WeChat and Miro application installers hosted on fake domains. They are prompted to execute a harmful script, mistakenly perceiving it as a legitimate security update. This bypasses recent macOS security measures that aimed to protect against older SHub attack methods.

The emergence of this variant matters as it underscores the evolution of malware to circumvent existing security protocols in macOS, highlighting that even newer systems like macOS can be targeted in sophisticated ways. It poses a significant threat to user privacy and financial data security, especially for those using crypto wallets and password managers. The persistence and file-hiding techniques make detection difficult, potentially leading to extensive data loss or misuse before users or security systems notice.

To address this threat, organizations should educate users on recognizing deceptive update prompts, particularly those involving unauthorized AppleScript usage. Enhanced monitoring of abnormal outbound traffic following Script Editor use and vigilance toward unauthorized LaunchAgents can aid in early detection. It would also be prudent for organizations to periodically review and update their security protocols involving application installations and keep systems regularly patched to close off exploits that might allow unverified code execution.
