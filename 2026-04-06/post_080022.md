Original Article: https://www.bleepingcomputer.com/news/security/hackers-exploit-react2shell-in-automated-credential-theft-campaign/

The article discusses a cybersecurity incident in which hackers exploited the React2Shell vulnerability in Next.js apps to automate credential theft on a large scale. By utilizing a framework named NEXUS Listener, attackers breached at least 766 hosts, primarily targeting cloud services, to collect sensitive data such as API keys, database credentials, and SSH private keys. This operation is attributed to the threat cluster UAT-10608, as identified by Cisco Talos researchers, who gained insights into the attack strategy by analyzing a publicly exposed instance of the NEXUS Listener.

This incident matters because it highlights the significant risks associated with software vulnerabilities, particularly when exploited in automated attacks targeting cloud infrastructures. The breach of sensitive credentials can lead to severe consequences, such as unauthorized access to cloud resources, data theft, financial fraud, and possible supply chain attacks. The exposure of sensitive data also heightens regulatory risks and privacy violations, given the breadth of compromised information and potential for further exploitation.

In response, system administrators should prioritize applying security patches to mitigate the React2Shell vulnerability in their Next.js applications. Proactive steps include auditing server-side data exposure, rotating credentials, and enforcing the least-privilege principle to limit access. Introducing secret scanning, deploying Web Application Firewalls (WAF) or Runtime Application Self-Protection (RASP), and adopting AWS IMDSv2 can further enhance security postures. Additionally, ensuring SSH key hygiene by using unique keys can prevent lateral movement within compromised networks.
