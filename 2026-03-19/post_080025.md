Original Article: https://www.bleepingcomputer.com/news/security/connectwise-patches-new-flaw-allowing-screenconnect-hijacking/

The article highlights a cryptographic signature verification vulnerability detected in ScreenConnect versions prior to 26.1, tracked as CVE-2026-3564, which could lead to unauthorized access and privilege escalation. This flaw allows a threat actor to extract ASP.NET machine keys and use them for session authentication, potentially leading to unauthorized actions within ScreenConnect. ConnectWise has patched this vulnerability by strengthening the protection of machine keys in version 26.1, but on-premises users need to manually upgrade to this version.

This vulnerability is significant because ScreenConnect is extensively utilized by MSPs, IT departments, and support teams, potentially affecting a broad range of organizational operations and sensitive data. The fact that attempts to misappropriate ASP.NET machine key material have been observed underscores the vulnerability's critical nature, although no active exploitation has been confirmed yet. Such vulnerabilities present a serious security risk, especially if they have been or could be leveraged by nation-state actors.

Organizations using ScreenConnect should prioritize updating their on-premises deployments to version 26.1 to safeguard against potential exploitation. Additionally, they should implement measures to secure configuration files and secrets, monitor logs for anomalous authentication actions, safeguard backups, and keep all software extensions current. Such actions will strengthen defense against unauthorized access and safeguard sensitive operational data, minimizing risk and enhancing cybersecurity posture.
