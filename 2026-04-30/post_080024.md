Original Article: https://www.bleepingcomputer.com/news/security/hackers-exploit-rce-flaws-in-qinglong-task-scheduler-for-cryptomining/

**1) What happened:** Hackers exploited two remote code execution (RCE) vulnerabilities in the Qinglong task scheduler to deploy cryptomining malware on developers' servers. These flaws, CVE-2026-3965 and CVE-2026-4047, allowed unauthorized access to protected endpoints by mismatching middleware authorization logic with routing behavior. The attack began in early February, preceding the public disclosure of the vulnerabilities. Once exploited, malicious actors could execute shell commands to download and run cryptominers, consuming significant server resources.

**2) Why it matters:** This incident underscores the constant threat posed by vulnerabilities in widely-used open-source tools, which can be leveraged to facilitate unauthorized cryptocurrency mining, leading to increased operational costs and reduced performance. For developers using Qinglong, the situation highlights the risk of failing to update software or adequately secure exposed endpoints. Furthermore, it exemplifies how quickly cybercriminals can exploit newly discovered flaws, emphasizing the importance of rapid detection and response to security threats.

**3) What actions should be taken:** Organizations using Qinglong should upgrade to the version addressing these vulnerabilities or apply necessary patches, particularly those included in PR #2941. It is prudent to conduct a thorough security audit to identify any unauthorized processes, such as cryptominers, on their systems. Regularly reviewing and updating middleware authorization protocols and ensuring proper configuration of server software can prevent similar issues. Additionally, enhancing monitoring systems to quickly detect anomalous CPU usage may provide early detection of similar intrusions.
