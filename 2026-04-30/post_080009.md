Original Article: https://www.bleepingcomputer.com/news/security/official-sap-npm-packages-compromised-to-steal-credentials/

**1) What happened:**  
Several official SAP npm packages used extensively in enterprise development were infiltrated in a sophisticated supply chain attack attributed to TeamPCP. Specifically, four packages were tampered with to include a malicious preinstall script, executing a payload designed to steal a broad range of credentials from developer environments and CI/CD systems. The stolen information included npm and GitHub tokens, SSH keys, cloud credentials from major providers, and CI/CD secrets. The attack further propagated by attempting to modify other packages and repositories using stolen credentials.

**2) Why it matters:**  
This incident underscores the vulnerability of the software supply chain, particularly with widely-used tools like npm packages. Compromises at this level can lead to widespread exposure of sensitive credentials, potentially enabling unauthorized access to numerous systems and data breaches across affected organizations. The attack could propagate further, exacerbating its impact. Given the critical role such packages play in enterprise environments, their compromise highlights the need for heightened vigilance and robust supply-chain security measures to safeguard against such insidious threats.

**3) What actions should be taken:**  
Organizations using these SAP npm packages should immediately audit their systems and environments to identify any usage of the compromised packages. Affected versions should be removed and replaced with secure versions after ensuring all dependencies are clean. It's also advisable to rotate any exposed credentials such as API keys, tokens, and secrets. Additionally, implementing stringent monitoring for unusual activities in development environments can help detect potential breaches early. Finally, strengthening supply chain security by employing automated tools to flag and block malicious package activities should be considered.
