Original Article: https://www.bleepingcomputer.com/news/security/popular-wordpress-redirect-plugin-hid-dormant-backdoor-for-years/

1) The Quick Page/Post Redirect plugin, used on over 70,000 WordPress sites, contained a dormant backdoor implanted five years ago that could execute arbitrary code. Detected by Austin Ginder, this malware was linked to versions 5.2.1 and 5.2.2, which included a self-update mechanism from a third-party domain, covertly pushing malicious updates. These updates caused a hidden backdoor to execute actions such as SEO spam, targeting logged-out users. The breach highlights vulnerabilities that may exist in common software utilities if not properly vetted and monitored.

2) This incident underscores the importance of vigilant software review, especially in widely-used plugins that provide crucial functionality. A backdoor in such a common component can have widespread effects, compromising sites that depend on this utility. It exemplifies how malicious actors exploit plugins to infiltrate web ecosystems, potentially affecting SEO and other site operations. For security teams and website administrators, this reveals the broader risks posed by third-party dependencies and the need for diligent monitoring and governance of software components.

3) To mitigate the risk, website administrators should remove the infected plugin and replace it with a clean version once it's available. Regular audits of installed plugins for unverified update mechanisms are encouraged to identify any similar vulnerabilities. Additionally, monitoring plugin directories for suspicions about code sourcing can prevent future incidents. Security teams might also consider implementing endpoint protection strategies and regular system audits to detect abnormal behavior rapidly. Encouraging developers behind plugins to follow secure coding practices and submit code for frequent verification would further protect their user base.
