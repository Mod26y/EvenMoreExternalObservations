Original Article: https://www.bleepingcomputer.com/news/security/wordpress-malware-campaign-hides-payloads-in-steam-profiles/

A recent WordPress malware campaign has infected nearly 2,000 websites by using Steam Community profile comments to hide command-and-control (C2) data. The attacker employs invisible Unicode characters to encode a payload that builds a URL directing to a malicious script, thereby bypassing traditional detection systems. This stealthy method uses Valve's platform to mask activities and evade scrutiny. The attack involves various stages, including initial infection via stolen credentials or vulnerable components, extracting encoded payloads from Steam, and implementing a backdoor that responds to crafted requests with an authentication cookie.

The significance of this incident lies in its innovative method of hiding malicious payloads using popular platforms like Steam, making detection extremely challenging. The invisible Unicode technique leverages normal-looking text to conceal malware, demonstrating sophisticated obfuscation tactics that can potentially affect WordPress site users widely. Given that WordPress is a prevalent platform for websites, the impact of unnoticed infections could be widespread, affecting website integrity and visitor security, and emphasizing the need for enhanced vigilance in cybersecurity practices.

In response, site owners should proactively monitor for unusual activity related to Steam URLs, unexpected JavaScript injections, and connections to suspicious domains like hello-mywordl[.]info. Regular security audits, including checking for invisible Unicode in comments and unusual cached messages, are advisable. Prompt restoration from clean backups is crucial if compromised. Additionally, implementing strong authentication mechanisms, regular theme/plugin updates, and enhanced server security measures could mitigate future risks. Engaging with ongoing threat intelligence could offer further insights into evolving threats and bolster defense strategies.
