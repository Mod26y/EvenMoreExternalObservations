Original Article: https://www.bleepingcomputer.com/news/security/microsoft-windows-powershell-now-warns-when-running-invoke-webrequest-scripts/

1) Microsoft has introduced a security feature in Windows PowerShell 5.1 to warn users when scripts using the Invoke-WebRequest cmdlet are executed. This feature addresses a high-severity vulnerability, CVE-2025-54100, by alerting users to potential risks of remote code execution when web content is downloaded and parsed. This update prompts users with a security warning, especially targeting enterprise environments that rely on PowerShell for automation, advising the use of safer script parsing with the '-UseBasicParsing' parameter.

2) The new security measure is significant as it addresses a critical vulnerability in systems that frequently use PowerShell scripts, a common practice in enterprise environments for automation. PowerShell scripts can be a vector for attacks if not properly secured, and this warning seeks to mitigate remote code execution risks. The update emphasizes the need for heightened awareness around script-based vulnerabilities, encouraging safer script execution practices and emphasizing security in automated systems.

3) Organizations should consider updating their PowerShell scripts to integrate the '-UseBasicParsing' parameter to protect against unwanted script execution. IT departments need to educate their teams about the new security prompt, ensuring that personnel are aware of the implications of their response actions. Additionally, IT admins should include the KB5074204 update in their routine system updates to benefit from this added security layer, minimizing the chances of script-related vulnerabilities within their networked environments.
