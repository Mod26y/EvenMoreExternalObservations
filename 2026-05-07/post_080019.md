Original Article: https://www.bleepingcomputer.com/news/security/critical-vm2-sandbox-bug-lets-attackers-execute-code-on-hosts/

A critical vulnerability was discovered in vm2, a widely-used Node.js sandboxing library, which permits attackers to escape the sandbox and execute arbitrary code on the host system. The issue, identified as CVE-2026-26956, particularly affects environments with Node.js 25, where specific features like WebAssembly exception handling and JSTag support are enabled. A proof-of-concept for this exploit has been released, underscoring the potential severity of the issue. This vulnerability arises from improper exception handling that allows host-side error objects to leak into the sandbox, letting attackers access Node.js internals.

This vulnerability is significant because vm2 is extensively used, with over 1.3 million weekly downloads from npm. Organizations use it to run untrusted JavaScript code securely. The flaw jeopardizes security by allowing attackers to execute arbitrary commands on host systems, potentially leading to data breaches or disruptions. The release of a proof-of-concept exploit increases the urgency as it shows how attackers can practically exploit this vulnerability. It highlights ongoing challenges in securely isolating untrusted code, and the necessity of proactive security measures in open-source components.

Users of the vm2 library are advised to upgrade to version 3.10.5 or later, with the latest being 3.11.2, to protect against the CVE-2026-26956 exploitation. Additionally, security teams should review their systems for any use of the affected library versions and patch accordingly. It's also crucial to enforce stringent access controls and privilege management to minimize potential exploitation risks. Moreover, consider adopting automated security validation tools that can detect and address such vulnerabilities proactively, helping reinforce security measures and readiness against future threats.
