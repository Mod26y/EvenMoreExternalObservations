The HTTP/2 Bomb is a novel DoS attack that exploits web servers using default HTTP/2 configurations by employing two DoS techniques together: HPACK compression and flow-control stalling. It enables a single machine, even a typical home computer, to render servers inaccessible quickly by causing them to allocate and hold large amounts of RAM. This attack was analyzed using servers like NGINX, Apache, and Microsoft IIS, showing remarkable speed and efficiency in draining server resources, rendering them inoperative.

This attack highlights a critical vulnerability in popularly used web servers, emphasizing the need for robust defenses against rapidly evolving threats. Its ability to exhaust server memory quickly with minimal data input stresses the importance of addressing security gaps and reassessing server configurations that use default protocols. Understanding and mitigating such threats are vital to maintaining the availability and performance of online services integral to business operations and user satisfaction.

In response to this threat, organizations should evaluate the exposure of their HTTP/2 configurations and consider deploying available patches and updates, such as those for NGINX and Apache. Until patches are available for Envoy and IIS, temporarily disabling HTTP/2 or using a proxy/firewall to manage headers could lessen the threat. Additionally, implementing custom server configurations and conducting regular system testing through simulations can improve detection and mitigate the risk posed by such attacks.
