Original Article: https://www.bleepingcomputer.com/news/security/over-100-chrome-extensions-in-web-store-target-users-accounts-and-data/

1) The article reports that over 100 malicious extensions in the Chrome Web Store are attempting to steal user accounts and data. These extensions, discovered by researchers at Socket, are involved in a coordinated campaign using common command-and-control infrastructure. The extensions, published under different identities, cover a variety of categories and are engaged in activities like session hijacking, data harvesting, and deploying backdoors. This campaign is potentially linked to a Russian malware-as-a-service operation, with extensions utilizing techniques to steal OAuth2 Bearer tokens, access personal Google account information, and manipulate user sessions.

2) This situation is critical as it poses a significant threat to user privacy and security, involving trust exploitation in the Chrome Web Store ecosystem. Stolen OAuth2 tokens can grant malicious actors access to users' data, allowing unauthorized actions on their behalf, potentially leading to data breaches and compromised accounts. The use of popular internet services as a disguise increases the risk, as users might unknowingly install these malicious extensions, thus widening the attack surface.

3) Users should immediately audit their Chrome extensions using the list of extension IDs provided by Socket and remove any suspicious ones. Organizations should enhance endpoint security measures, including runtime protection against unauthorized data access and account compromise. Google needs to expedite the removal of these malicious extensions to protect users. Raising awareness about the danger of installing extensions from unknown publishers can also reduce potential exposure. Admins should consider deploying browser security solutions that flag untrustworthy extensions across organizational devices.
