Original Article: https://www.bleepingcomputer.com/news/security/appsflyer-web-sdk-used-to-spread-crypto-stealer-javascript-code/

**1) What Happened:**  
The AppsFlyer Web SDK was temporarily hijacked in a supply-chain attack, where malicious JavaScript code was inserted to steal cryptocurrency by intercepting wallet addresses entered on applications using the SDK. This attack affected a large number of users due to AppsFlyer's widespread use in over 100,000 web and mobile applications. The obfuscated malicious script maintained normal SDK functions while stealthily replacing legitimate wallet addresses with those controlled by the attackers. While the security breach was identified and contained within a couple of days, its full impact is still under investigation.

**2) Why It Matters:**  
This incident is significant because it underscores the vulnerability of third-party components in the supply chain and the potential widespread impact of such breaches. As AppsFlyer is used globally, many businesses and their users were susceptible to financial losses, particularly in the rapidly growing cryptocurrency sector. This breach highlights the importance of securing the entire software supply chain and ensuring trust in widely deployed SDKs, as they can be a critical target for cybercriminals looking to exploit end-user trust and cause financial damage.

**3) What Actions Should Be Taken:**  
Organizations using the AppsFlyer SDK should immediately audit their systems, examining telemetry logs for suspicious API requests related to AppsFlyer, and revert to known-secure SDK versions if necessary. It is advisable to evaluate their entire software supply chain for similar vulnerabilities. Strengthening monitoring and incident response capabilities can aid in quickly identifying and mitigating such threats. Additionally, staying informed through regular updates from software vendors regarding security incidents and adopting a robust vulnerability management program will mitigate future risks. Communicating clearly with stakeholders about potential exposure is also crucial to maintain trust and transparency.
