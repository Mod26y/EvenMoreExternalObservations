Original Article: https://www.bleepingcomputer.com/news/security/sonicwall-sma1000-flaws-exploited-as-zero-days-to-push-custom-malware/

The article describes how threat actors exploited two vulnerabilities in SonicWall SMA1000 series appliances in zero-day attacks, enabling them to install custom malware. The issues, identified as CVE-2026-15409 and CVE-2026-15410, involve server-side request forgery (SSRF) and command injection flaws, respectively. Attackers leveraged these vulnerabilities to gain root access, install malware such as Sou5 and ORANGETAIL, and potentially access internal networks. This represents a significant breach of security for organizations utilizing these devices.

Addressing these vulnerabilities is critical as it highlights the ongoing risks associated with unpatched security weaknesses in widely-used devices like SonicWall's. The exploitation of these zero-day vulnerabilities emphasizes the need for rapid patch management and robust monitoring capabilities to detect and mitigate unauthorized access. Organizations must be vigilant due to the potential that attackers could maintain access to internal systems, leading to data breaches or service disruptions.

In response, organizations using SonicWall SMA1000 appliances should promptly apply the patches released in versions 12.4.3-03453 and 12.5.0-02835. Additionally, undertaking comprehensive security assessments, including vulnerability scanning and penetration testing, would help identify and rectify similar potential vulnerabilities. Enhancing monitoring for anomalous activities and refining incident response protocols would further bolster the security posture against such sophisticated threats.
