Original Article: https://www.bleepingcomputer.com/news/security/clawjacked-attack-let-malicious-websites-hijack-openclaw-to-steal-data/

**What happened:**  
A vulnerability named "ClawJacked" was discovered in the OpenClaw AI platform. This flaw allowed malicious websites to take control of a local OpenClaw instance by brute-forcing access via the WebSocket interface. The browser's cross-origin policies failed to block it because connections to localhost were exempt from restrictions, enabling a malicious site to authenticate itself by guessing passwords. Once access was gained, the attackers had full control and could execute commands, exfiltrate data, and manipulate AI agents.

**Why it matters:**  
OpenClaw’s widespread adoption makes it a critical target for cyberattacks, with this flaw potentially compromising user data, sensitive information, and system integrity. The ability for attackers to exploit this vulnerability easily and silently presents a severe risk for any organization using the platform, as it could be leveraged to orchestrate sophisticated attacks and data breaches, damaging trust, causing financial harm, and threatening operational continuity.

**What actions should be taken:**  
Organizations using OpenClaw should update to version 2026.2.26 or later to neutralize the ClawJacked vulnerability. Utilizing unique, strong passwords could further prevent brute-force attacks. Security teams should review access logs for unusual activity, consider implementing additional network-level controls, and closely monitor any related phishing or malware activity. Continuous evaluation of third-party software and diligent patch management will help mitigate similar vulnerabilities in the future.
