1) The article titled "Cordyceps: Mushrooming Malicious Pull Requests Threaten Developer Workflows" discusses an emerging cybersecurity threat where adversaries exploit the workflow of software developers by creating malicious pull requests. These pull requests appear benign but contain code that can introduce vulnerabilities or backdoors into software projects. This tactic targets collaborative software development platforms, where developers often review and accept code changes through pull requests.

2) The threat of malicious pull requests matters because it exploits the collaborative nature of modern software development, potentially compromising the integrity of widely-used software. As open-source software becomes integral to many applications, introducing vulnerabilities through trusted contributions can have widespread and severe implications, compromising the security of numerous systems and users globally. This attack vector represents a shift towards more sophisticated and targeted approaches to compromising software supply chains.

3) To mitigate the risk posed by malicious pull requests, organizations could implement additional verification and review processes for code contributions. Automated tools that scan for potentially harmful code before integration could help identify and block malicious submissions. Training developers to identify suspicious pull requests and establishing robust authentication processes for contributors can further reduce susceptibility. Additionally, fostering a culture of security awareness among development teams can empower individuals to remain vigilant against such threats.
