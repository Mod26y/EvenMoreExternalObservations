A large-scale cyber campaign is targeting a critical SQL injection vulnerability identified as CVE-2026-26980 in Ghost CMS versions 3.24.0 through 6.19.0. Attackers exploit this flaw to steal administrative API keys, enabling them to inject malicious JavaScript into targeted websites, which includes prominent institutions like Harvard and Oxford Universities. This attack is part of a wider ClickFix campaign where the malicious scripts present fake verification prompts to visitors, leading to potential malware infections via downloaded payloads.

This situation highlights the significance of timely patching and regular security updates. The exploitation of this vulnerability emphasizes the potential wide-ranging impact on various sectors, including education and technology, affecting the integrity and trustworthiness of affected websites. The attack chain showcases sophisticated tactics to engage users unwittingly, underlining the critical need for robust website security frameworks and awareness among users regarding phishing and social engineering.

To mitigate these threats, administrators should immediately upgrade Ghost CMS to version 6.19.1 or newer and rotate all previously used keys. It is crucial to review websites for any indicators of compromise, such as injected scripts, based on listings provided by XLab. Maintaining a 30-day record of admin API call logs is also recommended to enable detailed post-incident analysis, aiding in the detection of any unauthorized access and behavior. Continual vigilance and prompt response to security advisories are essential in safeguarding against future exploitation attempts.
