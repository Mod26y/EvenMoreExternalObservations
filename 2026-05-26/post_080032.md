The Kali365 phishing-as-a-service platform targets Microsoft 365 accounts through device code authentication, circumventing multi-factor authentication by hijacking OAuth session tokens. Distributed on Telegram since April 2026, it simplifies account compromise without requiring password or MFA theft. The method involves tricking users into entering generated authentication codes, granting attackers full account access.

This development signifies a worrying trend where low-skilled attackers gain advanced capabilities, threatening organizational data security by exploiting legitimate authentication flows. The increased adoption across various platforms indicates a wider potential impact, with attackers able to access multiple cloud applications, posing a critical risk to data integrity and confidentiality.

Organizations should consider restricting or blocking device code authentication via Conditional Access policies and thoroughly audit device code usages. Developing robust phishing awareness programs is crucial to mitigate social engineering risks. Additionally, it’s advisable to preserve and report phishing activities and enhance surveillance of unauthorized device registrations as a proactive defense measure.
