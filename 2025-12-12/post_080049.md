Original Article: https://www.bleepingcomputer.com/news/security/malicious-vscode-marketplace-extensions-hid-trojan-in-fake-png-file/

**1) What happened:**
A covert cyber campaign, starting in February, involved 19 malicious extensions targeting developers on the VSCode Marketplace. These extensions masked a trojan within a fake PNG file located in dependency folders. When users installed these extensions, a modified version of popular npm packages, such as 'path-is-absolute', executed harmful code through an obfuscated JavaScript dropper. This campaign represents a significant supply-chain attack risk on Microsoft's VSCode environment, which is a key tool for developers, exposing the potential vulnerabilities within widely trusted platforms.

**2) Why it matters:**
The incident highlights the vulnerabilities of supply-chain attacks in developer environments, which have immense downstream effects due to the widespread usage of VSCode. Such attacks can compromise a numerous of projects, lead to intellectual property theft, and potentially insert malicious code across multiple systems. It underscores the need for heightened scrutiny of extensions and dependencies, as even trusted environments like the VSCode Marketplace can become vectors for sophisticated cyber threats, necessitating vigilance from both developers and platform providers to safeguard software integrity.

**3) What actions should be taken:**
Users who installed these extensions should conduct comprehensive system scans for indicators of compromise. Developers should enhance their security hygiene by reviewing and verifying the source and contents of package dependencies, particularly those bundled with extensions, and prioritize using reputable publishers. Organizations should also consider integrating robust vulnerability scanning tools and supply-chain security solutions like those from ReversingLabs. Lastly, raising awareness and training among developer communities regarding the risks associated with third-party extensions can help prevent similar incidents.
