Original Article: https://www.bleepingcomputer.com/news/security/wordpress-core-wp2shell-rce-flaws-get-public-exploits-patch-now/

1) The "wp2shell" vulnerabilities (CVE-2026-63030 and CVE-2026-60137) in WordPress Core allow unauthenticated remote code execution (RCE) by exploiting two independent flaws in versions 6.9.x and 7.0.x. These flaws involve a REST API batch-route confusion and an SQL injection in the WP_Query, and can be combined in an attack chain. Public proof-of-concept exploits are now available, significantly heightening the potential for widespread attacks on millions of WordPress sites.

2) This incident is crucial because WordPress powers over 500 million websites, making any security vulnerability in its core a substantial threat to global internet security. The availability of public exploits worsens the situation, as it facilitates automated and widespread exploitation of affected sites, risking the integrity, confidentiality, and availability of vast amounts of data. This underscores the importance of maintaining updated systems and demonstrates the potential impact of vulnerabilities in widely used open-source software.

3) Organizations should prioritize updating their WordPress installations to versions 7.0.2 or 6.9.5 to mitigate the vulnerabilities. In situations where immediate patching is impossible, temporary measures such as using plugins to block anonymous REST API access or implementing specific WAF rules to block the identified API routes can reduce the risk of exploitation. Additionally, organizations using Cloudflare should ensure WAF protections are enabled. Continuous monitoring for signs of compromise and revisiting security incident response plans will also aid in addressing potential exploitation.
