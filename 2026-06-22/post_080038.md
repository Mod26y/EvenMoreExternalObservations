Original Article: https://www.bleepingcomputer.com/news/security/hackers-exploit-info-disclosure-bug-in-gravity-smtp-wordpress-plugin/

The cybersecurity incident involving the Gravity SMTP WordPress plugin revolves around hackers exploiting an information disclosure vulnerability, tracked as CVE-2026-4020. By exploiting an exposed REST API endpoint, attackers can perform unauthenticated GET requests to access sensitive data, such as API keys, email service credentials, and server configuration details. This flaw is particularly dangerous due to its ease of exploitation and potential for misuse of email service credentials or detailed site information for further attacks. The vulnerability has been addressed in the latest plugin version, 2.1.5.

This breach has serious implications due to the extensive data exposure and potential misuse by hackers. Unauthorized access to email credentials could lead to impersonation and data breaches. Revealed system information can also facilitate more targeted and damaging attacks. The widespread use of WordPress amplifies the risk, as a large number of sites could be affected, potentially leading to broader implications across sectors and organizations using the platform.

Organizations utilizing the Gravity SMTP plugin should promptly update to version 2.1.5 to mitigate vulnerability. Security teams should analyze server logs for signs of compromise, particularly looking for requests with '/wp-json/gravitysmtp/v1/tests/mock-data'. Additionally, adding the most common source IPs of exploit attempts to blocklists could reduce risk. Proactive monitoring and applying timely patches to other plugins, like Avada Builder, as recommended, should be part of ongoing security measures.
