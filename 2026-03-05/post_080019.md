Original Article: https://www.bleepingcomputer.com/news/security/mail2shell-zero-click-attack-lets-hackers-hijack-freescout-mail-servers/

The article discusses a critical vulnerability, CVE-2026-28289, in the FreeScout helpdesk platform that allows attackers to execute remote code without user interaction or authentication. This flaw, found by OX Security researchers, is patched as of version 1.8.207, and it was discovered that attackers could exploit this vulnerability by sending a specially crafted email to any address within FreeScout. A bypass using a zero-width space character allows malicious uploads, highlighting deficiencies in prior security fixes.

This vulnerability is significant as it could lead to full server compromise, data breaches, and network intrusions, potentially affecting an organization’s entire infrastructure. FreeScout is a widely used platform, indicated by thousands of GitHub stars and numerous public instances. The nature of this zero-click vulnerability, needing no user interaction, increases the risk of exploitation, posing a severe threat to organizations that fail to promptly update their systems.

Recommended actions include swiftly upgrading to version 1.8.207 for all FreeScout instances to mitigate the flaw. Additionally, organizations should consider disabling 'AllowOverrideAll' in Apache configurations to prevent exploitation pathways even in patched environments. Continuous security monitoring and educating IT staff on the potential risks and necessary updates are also prudent to bolster defenses against similar threats.
