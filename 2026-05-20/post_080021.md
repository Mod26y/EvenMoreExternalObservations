Original Article: https://www.bleepingcomputer.com/news/security/max-severity-flaw-in-chromadb-for-ai-apps-allows-server-hijacking/

The article discusses a high-severity vulnerability in the ChromaDB project, specifically affecting the Python FastAPI version. The flaw, CVE-2026-45829, allows attackers to execute arbitrary code by exploiting a poorly positioned authentication check on the API server. Roughly 73% of internet-exposed instances are vulnerable as the security issue remains unresolved, and communication attempts by researchers to the maintainers for a fix have been unanswered.

This vulnerability is significant because ChromaDB is integral to applications accessing semantically relevant documents during AI inference, and the flaw impacts nearly 14 million monthly downloads on PyPI. If left unchecked, it could lead to significant data breaches or manipulation as attackers could insert and execute malicious AI models into server processes, compromising broader AI deployments and potentially leading to widespread impacts.

To address this issue, users are advised not to expose the Python API server online and to consider using the Rust front-end instead. Limiting network access to the API port can also mitigate risks. Evaluating and scanning machine learning model artifacts before runtime is suggested to prevent executing untrusted code, especially when employing ‘trust_remote_code’ settings. Such proactive measures can help safeguard systems while awaiting an official patch for the vulnerability.
