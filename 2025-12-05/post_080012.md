Original Article: https://www.bleepingcomputer.com/news/security/hackers-are-exploiting-arrayos-ag-vpn-flaw-to-plant-webshells/

The article highlights a critical command injection vulnerability found in Array AG Series VPN devices, which hackers have exploited to install webshells and create unauthorized user accounts. This vulnerability was fixed in a May security update, yet the absence of an identifier has hindered tracking and patching efforts. The threats have been actively targeting organizations, particularly in Japan, since at least August, using a known IP address for communications. The vulnerability affects devices running ArrayOS AG 9.4.5.8 and earlier, which many enterprises rely on for secure remote access.

The significance of this vulnerability lies in its potential impact on enterprise networks that use these VPN devices for remote access. If successfully exploited, this could allow unauthorized access, data breaches, and could potentially compromise entire systems, impacting the confidentiality, integrity, and availability of organizational data. With a substantial number of affected devices globally, particularly in major markets like the US and Asia, the flaw underscores the importance of timely patch management and the risks of untracked vulnerabilities.

Organizations using the affected ArrayOS AG versions are encouraged to update their systems to version 9.4.5.9, which addresses the vulnerability. If an update is unfeasible, disabling the unused DesktopDirect features and implementing URL filtering to block risky URLs can be effective workarounds. Organizations should also be vigilant in monitoring for unusual activity, especially originating from the identified IP address, and engage with security vendors for further insights and protection tailored to this exploit. Proactively addressing such vulnerabilities can help prevent potential breaches and maintain robust cybersecurity defenses.
