Original Article: https://www.bleepingcomputer.com/news/security/fortibleed-campaign-used-custom-fortigate-sniffer-to-steal-credentials/

The FortiBleed campaign exploited Fortinet FortiGate devices globally, using a custom-built sniffer to illegally obtain authentication credentials. Through advanced methods like credential stuffing, brute-force attacks, and packet sniffing, attackers accessed and extracted vital authentication data from network traffic across numerous protocols. Utilizing a tool named "FortigateSniffer," threat actors monitored and harvested sensitive information, posing severe risks to affected organizations. This incident underscores the capabilities of infiltrators to convert diagnostic features intended for security maintenance into tools for malicious activities.

The significance of this attack lies in its scale and sophistication, affecting over 430,000 FortiGate firewalls globally. It exposes vulnerabilities within trusted, high-security network components that many organizations rely on to protect sensitive information. With attackers using advanced GPU resources to crack passwords, existing security measures, especially those relying on Fortinet infrastructure, are challenged, threatening the confidentiality and integrity of organizational data and communications.

To mitigate these risks, organizations using FortiGate devices should conduct thorough reviews of their systems and network configurations, focusing on signs of compromise linked to the published list of targeted IP addresses. Strengthening password policies and regularly updating configurations to close potential vulnerabilities can help safeguard against similar attacks. Furthermore, implementing enhanced logging, monitoring capabilities, and using multi-factor authentication can provide an added layer of security to detect and respond to unauthorized access attempts proactively.
