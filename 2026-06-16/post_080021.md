Original Article: https://www.bleepingcomputer.com/news/security/simplehelp-bug-lets-hackers-create-rogue-remote-support-accounts/

The article reports a critical vulnerability, CVE-2026-48558, in SimpleHelp remote management software, which allows unauthenticated attackers to create privileged technician accounts on servers using OIDC authentication. This vulnerability is present in SimpleHelp versions 5.5.15 and older, plus 6.0 pre-release versions. Unauthenticated attackers can bypass multi-factor authentication, gaining access to manage endpoints, execute scripts, and more. SimpleHelp has addressed this flaw in updates released on June 9, 2023.

This vulnerability is significant because it compromises the security and integrity of systems, allowing unauthorized access to sensitive operations and data in large enterprises using OIDC. The exploit's potential for unauthorized control over managed endpoints and script execution could lead to data breaches, operational disruption, and other cyber threats. With substantial exposure to the public internet, even a small percentage of affected servers represents a notable risk.

Organizations using SimpleHelp should upgrade to the latest versions that contain the patches for CVE-2026-48558. For those unable to update immediately, mitigating risks by using IP-based allowlists to limit trusted login sources can reduce the threat. Regularly reviewing server logs and searching for rogue accounts can help detect potential compromises. Continuing to monitor updates from SimpleHelp and guidance from security researchers ensures preparedness against future vulnerabilities and threats.
