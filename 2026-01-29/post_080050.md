Original Article: https://www.bleepingcomputer.com/news/security/new-sandbox-escape-flaw-exposes-n8n-instances-to-rce-attacks/

1) Two critical vulnerabilities, identified as CVE-2026-1470 and CVE-2026-0863, were found in the open-source workflow automation platform n8n by JFrog researchers. These vulnerabilities allow attackers to execute arbitrary code and potentially gain complete control over the platform instances. CVE-2026-1470 involves a JavaScript AST sandbox escape, while CVE-2026-0863 is a Python-based sandbox escape. Despite requiring authentication, the critical nature of these flaws is highlighted by their potential exploitation to achieve infrastructure-level control.

2) The discovery of these vulnerabilities is significant because n8n, with over 200,000 weekly downloads, is widely used to automate tasks and integrate with AI and large language model services. The flaws could enable attackers to access sensitive data and disrupt the workflow automation processes vital for business operations. Given the platform's widespread use and the critical severity of the vulnerabilities, organizations using n8n must be vigilant, as these security flaws could result in severe breaches.

3) Organizations using self-hosted n8n instances should upgrade their systems to versions 1.123.17, 2.4.5, and 2.5.1 for CVE-2026-1470 and versions 1.123.14, 2.3.5, and 2.4.2 for CVE-2026-0863 to mitigate the risks posed by these vulnerabilities. Additionally, organizations need to conduct regular security assessments and audits to ensure that any instances of n8n are properly secured and continuously patched. Educating users on the potential threats and implementing stricter authentication measures can further prevent unauthorized access and exploitation of these vulnerabilities.
