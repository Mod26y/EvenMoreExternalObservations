Original Article: https://www.bleepingcomputer.com/news/security/microsoft-rejects-critical-azure-vulnerability-report-no-cve-issued/

1) What happened: A security researcher identified a critical vulnerability in Microsoft's Azure Backup for AKS that allowed privilege escalation without prior Kubernetes permissions. Despite the researcher’s claims and CERT’s verification of the flaw, Microsoft dismissed the report as not being a security issue and denied a CVE issuance, insisting the vulnerability needed existing admin rights. Subsequent observations indicated a quiet fix by Microsoft, altering previous behavior but without public acknowledgment or user notification.

2) Why it matters: This situation exposes concerns about vendor transparency and accountability in vulnerability disclosures. The lack of a CVE and public advisory denies organizations the information necessary to assess their exposure and mitigate risks. It underscores a systemic issue where disputes between researchers and vendors could leave security gaps unaddressed, endangering customer environments while potentially eroding trust in vendor-managed security protocols.

3) What actions should be taken: Organizations using Azure services should conduct their security assessments focusing on role permissions and access configurations. Establishing independent vulnerability management processes can aid in identifying risks when vendor communication is lacking. Security teams could advocate for clearer disclosure frameworks that prioritize user protection over vendor discretion, and engage in community discussions to better navigate these industry challenges. Regularly reviewing external security advisories and maintaining continuous dialogue with vendors can help preemptively address future discrepancies.
