Original Article: https://www.bleepingcomputer.com/news/security/tycoon2fa-hijacks-microsoft-365-accounts-via-device-code-phishing/

1) The Tycoon2FA phishing kit has been updated to include device-code phishing attacks, targeting Microsoft 365 accounts through OAuth 2.0 device authorization grants. Despite a previous law enforcement disruption, the platform has resumed and enhanced its evasion techniques. Attackers send phishing emails with Trustifi click-tracking URLs, deceive users into providing OAuth tokens, and gain access to victim data by registering a rogue device on the victim's account, thus compromising sensitive information like emails and cloud files.

2) This development highlights an escalation in phishing sophistication and effectiveness, as the capability to hijack accounts through legitimate-looking processes poses a significant security risk. Organizations relying on Microsoft 365 could face substantial data breaches, loss of sensitive information, and operational disruption. The increasing occurrence of this attack type stresses the need for robust authentication and authorization practices in defending against evolving cyber threats within complex, cloud-based digital environments.

3) Organizations should consider disabling the OAuth device code flow when unnecessary, enforcing stricter control over OAuth permissions, and mandating admin approval for third-party applications. Implementing Continuous Access Evaluation (CAE) and compliant device access policies can further safeguard systems. Additionally, it is vital to regularly monitor Entra logs for suspicious activities, such as deviceCode authentication and unusual user agent activities, and keep up-to-date with threat intelligence, including indicators of compromise provided by cybersecurity researchers.
