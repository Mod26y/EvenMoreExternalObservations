Original Article: https://www.bleepingcomputer.com/news/security/credit-card-theft-campaign-abuses-stripe-to-host-stolen-payment-info/

**1) What happened:**
A credit card theft campaign has exploited Stripe's trusted API infrastructure to host and exfiltrate stolen payment data via malicious code delivered through Google Tag Manager containers. This campaign targets Magento/Adobe Commerce checkout pages to capture sensitive customer information like credit card details and personal identifiers. After stealing the data, it is obfuscated and stored as a fake customer record in the attacker's Stripe account, essentially using Stripe for storage and avoiding detection. This method bypasses typical security measures since stripe.com domains are generally considered safe.

**2) Why it matters:**
This campaign underscores the increasing sophistication of cyber threats targeting e-commerce platforms, leveraging trusted third-party services like Stripe and Google Tag Manager to bypass security defenses. It highlights a critical vulnerability in the way secure domains are implicitly trusted, potentially allowing significant data breaches. Such attacks jeopardize not only consumer security and privacy but also the trust and financial well-being of businesses relying on these platforms for transactions and service delivery.

**3) What actions should be taken:**
Businesses should enhance monitoring and verification of all third-party scripts and domain interactions, even those previously deemed secure. Implementing robust anomaly detection systems, revising existing Content Security Policies, and conducting regular security audits of third-party integrations can help mitigate risks. It is also advisable for users to utilize virtual cards with strict limits to minimize potential losses from such exposures. Additionally, continued education on threat landscapes and periodic updates to security protocols will be vital for traders and developers using platforms like Stripe and GTM.
