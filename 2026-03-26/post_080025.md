Original Article: https://www.bleepingcomputer.com/news/security/polyshell-attacks-target-56-percent-of-all-vulnerable-magento-stores/

The article reports on active cyberattacks targeting Magento Open Source and Adobe Commerce Installations, exploiting a vulnerability called 'PolyShell'. This issue primarily affects 56.7% of vulnerable stores, enabling attackers to achieve remote code execution or account takeover through the REST API. Although a fix was released in a beta version, it is not yet available in the stable branch. Additionally, a sophisticated WebRTC-based payment card skimmer has been identified, which can bypass standard security measures due to its advanced execution techniques and evade traditional security controls.

This incident is significant because the attacks exploit a widespread e-commerce platform, potentially compromising sensitive customer data and financial transactions. The novel skimmer method presents an advanced threat capable of evading security policies, increasing the risk of financial and reputational damage for affected businesses. Furthermore, the vulnerability's exploitation highlights the urgency of proper patch management and highlights potential delays in security updates.

Organizations using affected Magento versions should immediately implement security measures to protect against the PolyShell vulnerability. They can monitor for indicators of compromise and apply temporary workarounds or configurations to restrict file uploads in the REST API. Regularly updating security patches and maintaining direct communication channels with vendors for urgent fixes is also essential. Furthermore, reinforcing Content Security Policy rules and closely monitoring network traffic for suspicious WebRTC connections can help mitigate the risk of data exfiltration.
