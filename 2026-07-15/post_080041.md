Original Article: https://www.bleepingcomputer.com/news/security/nearly-300-github-repos-pose-as-legit-software-to-push-malware/

A significant security breach involved a threat actor creating nearly 300 fake GitHub repositories to distribute infostealer malware. This campaign targeted users searching for legitimate software, like cryptocurrency tools, developer utilities, and secure email providers, redirecting them to malicious download pages. The infostealer targeted data from web browsers, cryptocurrency wallets, and messaging apps, with exfiltrated data sent to a Russia-based command-and-control server. Although GitHub removed many malicious repositories, some redirecting pages remained active, indicating an ongoing threat to unwary users seeking free software downloads.

This incident is crucial as it highlights the persistent and evolving threat landscape on platforms like GitHub, illustrating how threat actors exploit trust and known brands to distribute malware. The campaign's ability to collect sensitive data swiftly without establishing persistence poses a significant risk to individuals and organizations alike. Moreover, the incident showcases a novel capability of bypassing Chrome's App-Bound Encryption, emphasizing the need for constant vigilance and improved user awareness regarding download legitimacy. It underscores the importance of secure software distribution practices, particularly for open-source platforms.

In response to this threat, organizations should update security training programs to educate users on recognizing phishing techniques and downloading trusted, verified software sources. Security teams may consider employing detection strategies, using tools like Yara rules shared by researchers to identify malicious activities. Improved monitoring and analysis of network traffic can also help detect unusual behaviors. Additionally, reinforcing partnerships with platforms like GitHub may lead to more robust mechanisms for detecting and eliminating malicious repositories, enhancing overall cybersecurity defenses.
