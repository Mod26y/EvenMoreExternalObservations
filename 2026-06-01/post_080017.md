Original Article: https://www.bleepingcomputer.com/news/security/palo-alto-globalprotect-vpn-auth-bypass-flaw-now-exploited-in-attacks/

The article discusses the exploitation of a vulnerability, identified as CVE-2026-0257, in Palo Alto Networks’ GlobalProtect VPN system. This vulnerability allows attackers to bypass authentication by exploiting improperly validated authentication override cookies, leading to unauthorized VPN connections. Initially rated medium, the vulnerability's severity was later elevated to high as it was actively exploited. Attackers, leveraging this flaw, collected public keys via HTTPS, enabling them to forge cookies that granted system access, targeting unpatched systems specifically.

This flaw's exploitation poses a significant threat to corporate network security, given GlobalProtect VPN's widespread use. Unpatched vulnerabilities can enable cybercriminals to access sensitive internal network resources, potentially leading to data breaches or other malicious activities. The inclusion of this flaw in CISA's Known Exploited Vulnerability catalog further underlines its potential impact. Protecting network perimeters via robust VPN systems is crucial for organizational cybersecurity, making addressing such vulnerabilities critical to prevent unauthorized access and data exposure.

Organizations using GlobalProtect should promptly apply the latest security patches to address this flaw. Administrators are advised to disable the authentication override feature if possible or use distinct certificates for these cookies separate from other services on the device. These measures help mitigate risks associated with this vulnerability. Regular security audits and updating systems according to advisories can enhance resilience against similar threats. Enabling robust network monitoring can also aid in detecting and responding to unauthorized access attempts quickly.
