Original Article: https://www.bleepingcomputer.com/news/security/wp-maps-pro-bug-exploited-to-create-admin-accounts-on-wordpress-sites/

1) **What Happened:** A critical security vulnerability (CVE-2026-8732) in the WP Maps Pro WordPress plugin allowed unauthorized users to create administrator accounts. Discovered by security researcher David Brown, it affected versions 6.1.0 and older, facilitating unnoticed entry into systems for hackers. This flaw was related to the plugin's “temporary access” feature, which was insufficiently protected against unauthorized use, enabling attackers to execute code that added rogue admin users.

2) **Why It Matters:** The exploitation of this vulnerability presents significant risks, as gaining admin access means attackers can alter site content, deploy malicious software, exfiltrate data, and establish persistent access. These activities could severely compromise websites, risking business operations and data privacy. Notably, the plugin is widely used, enhancing the potential impact on businesses dependent on WordPress for ecommerce, geography-based services, and content management.

3) **Actions to Take:** Website administrators using WP Maps Pro should upgrade to version 6.1.1 immediately, which contains the patch for this vulnerability. It's also advisable to audit logs for unauthorized access or unusual activity, remove any suspicious admin accounts, and employ additional security measures, such as Web Application Firewalls (WAFs), to mitigate future exploitation attempts. Regular plugin updates and security scans can help prevent similar vulnerabilities from being exploited.
