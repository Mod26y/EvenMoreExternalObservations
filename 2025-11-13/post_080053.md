Original Article: https://www.bleepingcomputer.com/news/microsoft/microsoft-fixes-bug-causing-false-windows-10-end-of-support-alerts/

Microsoft recently addressed a bug that caused Windows 10 users to receive inaccurate end-of-support notifications. These alerts affected systems enrolled in the Extended Security Updates (ESU) program and versions still supported until the late 2020s. Although these messages regarding support status were incorrect, affected devices continued receiving crucial security updates. Microsoft implemented a cloud update and released a specific security update, KB5068781, to correct this error across impacted systems.

This issue carries significance as it can cause confusion among users and administrators about the support status of their systems, potentially leading them to believe their critical infrastructure is unsupported. Such confusion could lead to unnecessary expenditure on upgrades or new installations. Additionally, if not addressed, it could create security risks should organizations fail to apply necessary security patches, mistakenly assuming their systems are out of support.

Organizations are encouraged to ensure all their systems receive the KB5068781 update to resolve false messaging. For systems unable to immediately install the update, administrators have the option of implementing a Known Issue Rollback (KIR) group policy. It's crucial to assess existing configurations, such as connectivity and firewall settings, to ensure they don't inadvertently block the distribution of crucial updates. Continuous monitoring and timely application of patches will help maintain robust security postures.
