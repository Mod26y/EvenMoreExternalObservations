Original Article: https://www.bleepingcomputer.com/news/security/hackers-are-exploiting-a-critical-litellm-pre-auth-sqli-flaw/

1) The article reports a critical pre-authentication SQL injection vulnerability, identified as CVE-2026-42208, in the LiteLLM, an open-source LLM proxy. This flaw is being actively exploited by attackers who can bypass authentication by crafting specific 'Authorization' headers, allowing them to read and modify database information. The database houses sensitive data, including API keys and credentials, which can be exploited for unauthorized access. The issue began seeing exploitation roughly 36 hours post-disclosure, with attackers conducting carefully crafted payloads to target sensitive tables without testing less critical ones, indicating a focused campaign.

2) This vulnerability is significant because it affects a widely used open-source project, LiteLLM, which has substantial implications for developers and platforms managing AI models. The ability to exploit a pre-auth SQL injection flaw without authentication opens avenues for attackers to access and manipulate sensitive information potentially leading to unauthorized system access or further breaches. Additionally, the recent history of a supply chain attack on LiteLLM compounds its vulnerability making it a high-priority target for malicious actors interested in sensitive credentials that could be employed in broader cybersecurity attacks.

3) In response to this information, organizations that use the LiteLLM should upgrade to version 1.83.7 or later to remediate the SQL injection flaw. Immediate rotation of all virtual API keys, master keys, and provider credentials stored in vulnerable LiteLLM instances should be undertaken to mitigate potential compromises. Those unable to upgrade should consider implementing the suggested workaround of disabling error logs to limit exposure. Furthermore, monitoring systems for unusual access patterns or any database query anomalies could help identify ongoing exploitation attempts and enhance the incident response efforts.
