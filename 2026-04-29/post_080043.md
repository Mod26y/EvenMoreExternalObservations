Original Article: https://www.bleepingcomputer.com/news/security/checkmarx-confirms-lapsus-hackers-leaked-its-stolen-github-data/

1) The Checkmarx incident involved the LAPSUS$ hacking group leaking stolen data from Checkmarx's private GitHub repositories. The breach resulted from a supply-chain attack facilitated by stolen credentials obtained from the Trivy incident. These credentials allowed the attackers to inject malicious code and publish harmful Docker images and extensions for Checkmarx's KICS security scanner, gathering sensitive data like credentials and config files. The leaked data, confirmed to have been published by LAPSUS$, originated from this compromise, although no customer data has been reported as compromised.

2) This incident is significant because it highlights the growing threat posed by supply-chain attacks and the persistence of advanced threat groups like LAPSUS$. The breach underscores vulnerabilities in using third-party tools and repositories, which can become vectors for widespread cyber-attacks. The potential exposure of sensitive development artifacts can jeopardize organizational data integrity and trust, emphasizing the need for enhanced security measures in managing access to critical infrastructure such as GitHub repositories.

3) Organizations should strengthen access controls and monitoring around development environments and supply chains to mitigate similar threats. This includes implementing multi-factor authentication, regularly rotating credentials, and using tools to detect unauthorized code changes. Conducting thorough security audits and patches for third-party dependencies can help reduce vulnerabilities. Organizations should also ensure a robust incident response plan is in place, facilitating swift action when breaches occur and keeping stakeholders informed to maintain transparency and trust.
